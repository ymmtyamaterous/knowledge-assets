## 機能提案一覧

### A. 既存仕様の未実装（優先度順）

---

#### A-1. 次のレッスンへのナビゲーション

**カテゴリ**: C（コンテンツ管理）に付随するUX改善  
**概要**: レッスンページ下部に「次のレッスンへ →」リンクを表示する。現在は `← 戻る` ボタンしかなく、学習フローが途切れやすい。  
**実装スコープ**: フロントエンドのみ  
**詳細**:
- レッスンページ (`/lessons/[id]`) でセクション内のレッスン一覧を取得し、現在のレッスンの次のレッスンIDを特定
- 次のレッスンが存在する場合は「次のレッスンへ →」ボタンを表示
- セクションの最後のレッスンの場合は「コース一覧へ戻る」を表示
- 完了ボタンとセットで学習フローを完結させる

---

#### A-2. クイズ解答履歴ページ（Q-05）

**カテゴリ**: Q（クイズ）  
**概要**: ユーザーが過去に受けたクイズの結果スコアと日時を一覧で確認できるページを追加する。データは `user_quiz_results` テーブルに既に保存されており、APIも実装済み（`GET /api/v1/users/me/quiz-results`）。フロントエンドのUI実装のみ必要。  
**実装スコープ**: フロントエンドのみ  
**詳細**:
- 配置場所：プロフィールページ内（`/profile`）のセクションとして追加 or 独立ページ（`/profile/quiz-results`）
- 表示内容：クイズ名（またはレッスン名）・スコア（例：`4 / 5`）・正答率・受験日時
- スコアに応じてバッジ色を変える（例：80%以上はピンク、60%以上は黄、それ以下はグレー）

---

#### A-3. 用語詳細ページ（G-02）

**カテゴリ**: G（用語辞典）  
**概要**: 現在は用語辞典ページにすべての用語がカード形式で表示されるのみで、個別の詳細ページが存在しない。用語ごとに詳細ページ（`/glossary/[id]`）を追加し、関連レッスンへのリンクも表示する。  
**実装スコープ**: バックエンド（既存API流用）+ フロントエンド  
**詳細**:
- URL: `/glossary/[id]`
- 表示内容：用語名・読み方・定義（Markdown対応）・タグ一覧・関連レッスンリンク
- API: `GET /api/v1/glossary/{id}` は実装済み（追加実装なし）
- 用語辞典一覧の各カードからリンクを張る

---

#### A-4. パスワード変更（U-06）

**カテゴリ**: U（ユーザー管理）  
**概要**: プロフィールページに現在のパスワードを確認した上でパスワードを変更する機能を追加する。  
**実装スコープ**: バックエンド + フロントエンド  
**詳細**:

**バックエンド**
- エンドポイント: `PUT /api/v1/users/me/password`
- リクエスト: `{ "currentPassword": "...", "newPassword": "..." }`
- バリデーション：現在のパスワードと一致するか確認 → 新しいパスワードを bcrypt でハッシュ化して保存
- レスポンス: `200 OK` or `400 Bad Request`（パスワード不一致）

**フロントエンド**
- プロフィールページ (`/profile`) に「パスワード変更」フォームを追加
- フォーム内容：現在のパスワード / 新しいパスワード / 確認用パスワード
- クライアント側でも `新しいパスワード === 確認用パスワード` を検証する

---

#### A-5. 達成バッジ機能（P-04）

**カテゴリ**: P（学習進捗）  
**概要**: セクションまたはコースのすべてのレッスンを完了した際に、達成バッジを付与する。`badges` / `user_badges` テーブルはDBスキーマに既に定義されている。  
**実装スコープ**: バックエンド + フロントエンド  
**詳細**:

**バックエンド**
- レッスン完了時（`POST /api/v1/lessons/:id/complete`）のユースケース内で、セクション/コースの全レッスン完了を判定し、条件を満たす場合はバッジを付与するロジックを追加
- バッジ一覧 API: `GET /api/v1/users/me/badges`
- `domain.Badge`・`domain.UserBadge` モデルを追加
- `BadgeRepository`, `BadgeUseCase` を実装

**フロントエンド**
- プロフィールページにバッジ一覧を表示
- バッジ取得時にトースト通知（「🎉 バッジを取得しました！」）を表示
- 取得済みバッジはカラー、未取得はグレースケールで表示

---

#### A-6. 模擬試験 専用UI（Q-03）

**カテゴリ**: Q（クイズ）  
**概要**: `is_mock_exam = true` のクイズ向けに、制限時間付きの模擬試験専用画面を追加する。現在は通常クイズと同じ画面で表示されている。  
**実装スコープ**: フロントエンドのみ（バックエンドのデータは既存の `Quiz` モデルを流用）  
**詳細**:
- 模擬試験開始前に「制限時間 XX 分」「全 XX 問」を確認する開始画面を表示
- カウントダウンタイマーを右上に常時表示
- 時間切れ時は自動的に回答を送信（未回答は不正解として扱う）
- 全問を一度に表示するか1問ずつかを確認する（通常クイズとの差異）
- 結果画面では正答率・かかった時間・問題ごとの正誤を表示

---

#### A-7. コンテンツ検索（C-05）

**カテゴリ**: C（コンテンツ管理）  
**概要**: キーワードによるレッスン・用語の横断検索機能を追加する。  
**実装スコープ**: バックエンド + フロントエンド  
**詳細**:

**バックエンド**
- エンドポイント: `GET /api/v1/search?q={keyword}`
- 検索対象: `lessons.title`, `lessons.content`, `glossary_terms.term`, `glossary_terms.definition`
- レスポンス例:
  ```json
  {
    "lessons": [ { "id": "...", "title": "...", "sectionId": "..." } ],
    "terms":   [ { "id": "...", "term": "...", "reading": "..." } ]
  }
  ```
- PostgreSQL の `ILIKE` または `to_tsvector` を使用

**フロントエンド**
- ヘッダーに検索バーを追加（虫眼鏡アイコンクリックで展開）
- 検索結果ページ (`/search?q=...`)：レッスン結果・用語結果をタブ or セクション分けで表示

### B. 新規提案機能

---

#### B-1. 学習連続日数（ストリーク）機能

**カテゴリ**: P（学習進捗）の拡張  
**概要**: 毎日学習した連続日数（ストリーク）をトップページまたはプロフィールページに表示し、継続学習のモチベーションを高める。  
**実装スコープ**: バックエンド + フロントエンド  
**詳細**:

**バックエンド**
- エンドポイント: `GET /api/v1/users/me/streak`
- ロジック：`user_lesson_progress.completed_at` の日付を集計し、今日から遡って連続した日数をカウント
- レスポンス: `{ "currentStreak": 7, "longestStreak": 21, "lastStudiedAt": "2026-03-26" }`

**フロントエンド**
- トップページのウェルカムセクションに「🔥 7日連続学習中！」と表示
- プロフィールページにも統計として表示

---

#### B-2. 学習カレンダー（ヒートマップ）

**カテゴリ**: P（学習進捗）の拡張  
**概要**: GitHub の Contribution グラフのように、過去数ヶ月の学習実績をカレンダー形式で可視化する。継続学習を視覚的に確認でき、習慣化を促す。  
**実装スコープ**: バックエンド + フロントエンド  
**詳細**:

**バックエンド**
- エンドポイント: `GET /api/v1/users/me/calendar?year={yyyy}`
- ロジック：`user_lesson_progress.completed_at` を日付単位で集計し、各日のレッスン完了数を返す
- レスポンス: `{ "days": [ { "date": "2026-03-26", "count": 3 } ] }`

**フロントエンド**
- 進捗ページ (`/progress`) の下部に学習カレンダーセクションを追加
- 完了数に応じたピンクの濃淡でセルを着色（0: グレー、1+: 薄ピンク → 濃ピンク）

---

#### B-3. メモのエクスポート機能

**カテゴリ**: U（ユーザー）の拡張  
**概要**: プロフィールページの「メモ一覧」から、すべてのメモを Markdown ファイルとしてダウンロードできるボタンを追加する。学習のまとめをローカルに保存・活用できる。  
**実装スコープ**: フロントエンドのみ  
**詳細**:
- 「メモをエクスポート（.md）」ボタンをプロフィールページのメモ一覧上部に配置
- クリック時に、各メモを `# レッスン名\n\n{content}\n\n---` 形式で結合した `.md` ファイルを `<a download>` で保存
- バックエンド変更なし（既存の `fetchMyNotes()` APIを活用）

---

#### B-4. 学習統計サマリ

**カテゴリ**: P（学習進捗）の拡張  
**概要**: プロフィールページに学習に関する統計情報をカード形式でまとめて表示する。  
**実装スコープ**: バックエンド + フロントエンド  
**詳細**:

**表示する統計情報**
| 統計項目 | 内容 |
|---------|------|
| 完了レッスン数 | 累計で完了したレッスンの総数 |
| 学習日数 | 1回以上学習した日の合計数 |
| メモ件数 | 保存済みメモの総数 |
| クイズ正答率 | 全クイズ結果の平均正答率 |

**バックエンド**
- エンドポイント: `GET /api/v1/users/me/stats`
- 既存の `user_lesson_progress`, `user_quiz_results`, `user_notes` を集計して返す

**フロントエンド**
- プロフィールページ上部に 2×2 のグリッドでカード表示
- ローディング中はスケルトンスクリーン
